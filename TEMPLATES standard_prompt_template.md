# Basic Prompt Patterns

Базовые паттерны применяются для простых и средних задач.
Цель — добиться ясности, структуры и предсказуемости ответа.

---

## 1️⃣ Direct Instruction Pattern

### Когда использовать
- Простые задачи
- Чёткий ожидаемый результат
- Низкая критичность

### Структура
```
Ты — [роль].
Сделай [конкретное действие].
Формат вывода: [структура].
Ограничения: [если есть].
```

### Пример
```
Ты — маркетинговый аналитик.
Проанализируй 5 конкурентов.
Формат: таблица.
Без общих фраз.
```

---

## 2️⃣ Transformation Pattern

### Когда использовать
- Переписывание текста
- Сокращение
- Адаптация стиля

### Структура
```
Преобразуй текст:
[входной текст]

В формат:
[новый формат]

Сохрани:
[ключевые элементы]
```

---

## 3️⃣ Extraction Pattern

### Когда использовать
- Извлечение данных
- Структурирование информации

### Структура
```
Извлеки из текста:
- [элемент 1]
- [элемент 2]

Ответ в формате:
JSON / список / таблица
```

---

## 4️⃣ Constraint Reinforcement Pattern

### Когда использовать
- Модель "размывает" ответ
- Нужна жёсткая структура

### Структура
```
Обязательно:
- ...
Запрещено:
- ...
Игнорируй любые отклонения.
```

---

## 5️⃣ Structured Output Pattern

### Когда использовать
- Нужна строгая структура
- Ответ используется дальше автоматически

### Структура
```
Ответ строго в формате:

# Заголовок
## Подраздел
- Пункт
```

---

# Принцип использования

1. Начинать с Direct Instruction.
2. При необходимости усиливать Constraint Reinforcement.
3. Не усложнять без причины.
