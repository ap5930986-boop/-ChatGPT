# Prompt Testing Playbooks

Этот документ описывает сценарии тестирования промптов.

---

## 1️⃣ Smoke Test

Цель:
Проверить, что промпт вообще работает.

Метод:
- Запустить 1–2 раза
- Проверить формат
- Проверить базовую полноту

---

## 2️⃣ A/B Test

Цель:
Определить более эффективную формулировку.

Метод:
1. Создать две версии промпта.
2. Запустить по 3 раза каждую.
3. Оценить по evaluation_metrics.md.
4. Выбрать лучшую.

---

## 3️⃣ Stress Test

Цель:
Проверить устойчивость к сложным входным данным.

Метод:
- Увеличить объём текста
- Добавить противоречивые данные
- Добавить шум
- Проверить устойчивость структуры

---

## 4️⃣ Edge Case Test

Цель:
Проверить граничные случаи.

Примеры:
- Очень короткий ввод
- Очень длинный ввод
- Некорректный ввод
- Двусмысленные инструкции

---

## 5️⃣ Regression Test

Цель:
Проверить, не ухудшилось ли качество после изменений.

Метод:
- Сохранить контрольные кейсы
- Повторно прогнать после обновлений
- Сравнить метрики

---

# Рекомендованный цикл

1. Создание промпта
2. Smoke Test
3. A/B Test (если критично)
4. Stress Test
5. Финализация
