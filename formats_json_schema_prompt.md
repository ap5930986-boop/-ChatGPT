# How to Fill and Maintain This Repository

## 1️⃣ Не добавляй статьи — добавляй инструменты

Репозиторий должен содержать:
- шаблоны
- паттерны
- готовые структуры
- проверенные примеры

Не хранить длинные теоретические тексты без практической ценности.

---

## 2️⃣ Каждый файл должен отвечать на 3 вопроса

1. Для чего это используется?
2. Когда применять?
3. Как выглядит пример?

Если нет этих трёх элементов — файл неполный.

---

## 3️⃣ Избегать дублирования

Перед добавлением нового файла:
- Проверь, нет ли похожего.
- Улучши существующий вместо создания нового.

---

## 4️⃣ Разделяй уровни сложности

Basic → для типовых задач  
Structured → для аналитики  
Advanced → для сложных систем

Не смешивать уровни в одном файле.

---

## 5️⃣ Тестируй перед коммитом

Перед добавлением:
- Запусти промпт минимум 3 раза
- Оцени по evaluation_metrics.md
- Проверь устойчивость структуры

Если средний балл < 4.0 — доработать.

---

## 6️⃣ Минимизируй токены

Удалять:
- повторения
- избыточные формулировки
- абстрактные описания

Добавлять:
- чёткие ограничения
- формат вывода
- конкретные критерии

---

## 7️⃣ Принцип архитектуры

Этот репозиторий — не сборник идей.
Это инженерная система.

Любой файл должен:
- Повышать предсказуемость модели
- Повышать структурированность ответа
- Снижать вариативность
- Уменьшать токен-шум
