# Prompt Evaluation Metrics

Этот документ определяет стандарты оценки качества промптов и ответов модели.

Цель — сделать качество измеримым, а не субъективным.

---

## 1️⃣ Полнота (Completeness)

Вопрос:
Ответил ли результат на все аспекты задачи?

Проверка:
- Нет ли пропущенных пунктов?
- Учтены ли ограничения?
- Соответствует ли цели?

Оценка: 1–5

---

## 2️⃣ Точность (Accuracy)

Вопрос:
Корректны ли факты и логика?

Проверка:
- Нет ли галлюцинаций?
- Логически ли обоснованы выводы?
- Соответствуют ли данные исходному контексту?

Оценка: 1–5

---

## 3️⃣ Структурированность (Structure Quality)

Вопрос:
Ясна ли структура?

Проверка:
- Есть ли заголовки?
- Логично ли распределены блоки?
- Нет ли хаотичных переходов?

Оценка: 1–5

---

## 4️⃣ Токен-эффективность (Token Efficiency)

Вопрос:
Есть ли лишняя вода?

Проверка:
- Есть ли повторения?
- Можно ли сократить без потери смысла?
- Есть ли избыточные объяснения?

Оценка: 1–5

---

## 5️⃣ Соответствие формату (Format Compliance)

Вопрос:
Ответ соответствует запрошенному формату?

Проверка:
- Таблица действительно таблица?
- JSON валиден?
- Markdown корректен?

Оценка: Да / Нет

---

## 6️⃣ Устойчивость (Stability)

Вопрос:
Повторный запуск даёт схожий результат?

Проверка:
- Нет ли сильной вариативности?
- Не теряется ли структура?

Оценка: Высокая / Средняя / Низкая

---

# Итоговая формула

Общая оценка = среднее арифметическое 1–4 критериев.

Минимальный проходной балл: 4.0
